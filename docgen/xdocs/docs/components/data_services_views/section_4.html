Issue: Substitutability.
This organization of the source code only allows for design-time substitutability. That is, the Data Services Views is a logical component that can be replaced during design by another logical component that conforms to the same specification. In fact, because the component's realizations manifest in the shoola.jar and are embedded with the data services, we have no run-time substitutability. This issue has to be considered when addressing the same substitutability issue for the data services (should be factored out in a separate jar) and we'll have to decide on the granularity of physical components — is the Data Services View going to be a separate jar from the Data Services or they will both end up in the same jar?
Issue: Dependency on Swing.
The BatchCallMonitor uses SwingUtilities to relay events within the UI thread. This dependency on Swing has to be removed, obviously. In fact, this is something the EventBus should take care of. However, the EventBus doesn't support dispatching of events to a given listener yet.
