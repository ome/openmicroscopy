Provides web components for OMERO services. The control flow is as follows:
  1. Requent is delivered to Spring's Dispatcher Servlet (see web.xml)
  2. DispatcherServlet uses Spring configuration to find the matching controller
  3. The controller uses services available in the Spring ApplicationContext,
  4. which use DAOs also in the context to access the Database
  5. Values to be returned are stored in a model and returned to a View with a implementation-independent name,
  6. which renders the values and returns them in the HTTP response

Important to note is that many technologies play a role in each of these steps. For example in the view step, Tiles (http://TODO) inserts various pieces of the response from its own controller
