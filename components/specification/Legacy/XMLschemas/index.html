Introduction

This document outlines the legacy XML Schemas of the Open Microscopy Environment Group. Detailed information on the current schemas and future development plans are available at http://www.openmicroscopy.org/site/support/file-formats. Further information on the groups work is available at http://www.openmicroscopy.org/

Warning

This document describes the legacy XML Schemas released before June 2007. There is no further development work taking place on the data model impemented by these files. These schemas were used by the original OME server.

Information on the new schemas is available on http://www.openmicroscopy.org/site/support/file-formats and the new schemas' namespace is located at http://www.openmicroscopy.org/Schemas/.

Schemas

This has the schema name, the current version, a link to the XSD file, and a short description

  • AnalysisChain - 2003 Version RC1 [AnalysisChain.xsd] The Analysis Chain schema. Analysis chains are how module outputs are connected to inputs of other modules.
  • AnalysisModule - 2003 Version RC1 [AnalysisModule.xsd] The Analysis Module Library schema. Analysis modules are image analysis algorithms or, more generally, how information is generated in OME. Makes use of CLI and MLI schemas.
  • BinaryFile - 2003 Version RC1 [BinaryFile.xsd] The Binary File schema is used to describes a file location, or the location of a fragment within a file.
  • CA - 2003 Version RC1 [CA.xsd] The standard custom attribute schema.
  • CLI - 2003 Version RC1 [CLI.xsd] The Command Line Interface sub-schema.
  • DataHistory - 2003 Version IR3 [DataHistory.xsd] The Data History schema. Data history describes the data dependency in OME.
  • MLI - 2003 Version IR2 [MLI.xsd] The Matlab Interface sub-schema.
  • OME - 2003 Version FC-TIFF [ome.xsd] The main schema which defines the OME ontology for microscopy. This schema makes use of all of the others. The FC-TIFF version is the same as the FC version but with the addition of the TiffData block needed to support OME-TIFF. This minor update was made in September 2007.
  • STD - 2003 Version RC2 [STD.xsd]The Semantic Type definition schema. Once a Semantic Type is defined, data for the defined type can be imported into the OME server.

Note: Some browsers will try to render XSD files when you view them. This can result in either a blank screen or unformatted text. Choose to either download the file or view the source.
